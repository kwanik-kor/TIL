# 1부 AI 프로젝트를 시작하며: 기본 점검 

## 01. 프로젝트가 실패할 확률을 추산해보았나?

2024년 S&P 글로벌 마켓 인텔리전스(Global Market Intelligence)
- AI 프로젝트의 42% 완전 폐기
- 가트너(Gartner) 27년까지 에이전틱 AI 프로젝트의 40% 이상이 취소될 것이라는 전망

**생성형 AI/LLM을 이용한 프로젝트가 인기가 많은 이유**
- 자연어를 이용하기 떄문
- 대규모 학습이 필요 없는 제로샷(zero-shot) 또는 퓨샷(few-shot) 프롬프팅으로 무언가를 할 수 있음
- 모델이 범용으로 쓰일 수 있음
- 멀티 모달(multi modal)이 됨
	- 입출력 강화에 따라, 텍스트 출력 뿐 아니라 이미지 생성 등 다양한 형태의 데이터에 접근이 가능해짐
- 복잡한 일을 덜어낼 수 있음

**생성형 AI/LLM 프로젝트들의 치명적 함정들**
- 데이터가 편향될 수 있음
- 평가 지표를 잘못 선택할 수 있음
	- BLEU(Bilingual Evaluation Understudy): 생성 텍스트 품질 평가지표, 생성물이 참조문과 얼마나 많은 n그램을 정확히 일치시키는가
	- ROUGE(Recall-Oriented Understudy for Gisting Evaluation): 텍스트 요약이나 번역 품질 평가 지표, 생성된 텍스트가 사람이 만든 정답과 얼마나 많은 단어나 구절을 공유하는지 측정
	- + 실제 일을 주고 일의 결과를 가지고 점수를 내는 방식을 사용해야 함
- 환각 현상
- 데이터의 법적 문제

## 02. 생성 AI 기술은 절대반지일까?

> 뉴질랜드 팩엔세이브(Pak'nSave)의 잘못된 레시피 추천
> 뉴욕시 비즈니스 상담 챗봇의 부적절한 상담내용

**기존 머신러닝/규칙기반 기술이 더 적합한 문제도 있다**
- 정밀한 수치 계산이 필요한 경우
	- 패턴 기반 추론을 사용함에 따라 복잡한 수치 연산에서는 상대적으로 더 높은 오차율을 보임
	- 절대적인 정확성이 필요한 경우 팩트체크 서비스를 이용해 가드레일을 구축하거나 사람 리뷰를 거치게 하는 편이 안전함
		- 구글 팩트체크(Google FactCheck)
		- 이머전트(Emergent): 실시간 미확인 정보, 소문 추적/검증 플랫폼
- 의사결정 지원 시스템(DSS, Decision Support System)
	- 병원에서 약물 부작용 및 중복 처방에 대한 경고를 보내는데, 이 경우 규칙기반 시스템이 더 나을 수 있음
